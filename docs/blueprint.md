# **App Name**: HealthSphere AI

## Core Features:

- AI Disease Prediction Dashboard: A dashboard UI for users to input structured health data (clinical parameters) and receive risk predictions (e.g., Heart, Diabetes, Kidney, Breast Cancer) from backend ML models, along with clinical recommendations.
- Medical Image Analysis Interface: An interface for uploading medical images (e.g., Chest X-rays) for deep learning-based analysis, displaying detection results (e.g., Pneumonia), and associated recommendations.
- Interactive AI Health Assistant: A chat interface powered by an LLM-style API where users can describe symptoms or ask medical questions, receiving explanations, general guidance, and suggestions for appropriate specialists.
- Personalized Health Summary Display: A dedicated dashboard summarizing all risk predictions, medical image analysis results, and AI assistant interactions for the current session.
- Nearby Hospital Locator: A UI feature allowing users to find and view nearby hospitals using their current location or a search input, integrated with Google Maps.
- Downloadable Patient Report: Functionality to generate and download a comprehensive patient health report in PDF format, compiling all predictions, analyses, and recommendations from the current session.

## Style Guidelines:

- Primary color: A deep, professional blue (#1F43AA) to evoke trust and advanced technology, contrasting effectively with light backgrounds.
- Background color: A very light, desaturated blue-grey (#F2F4F7) to provide a clean and calm canvas for detailed information.
- Accent color: A moderate indigo-purple (#6C4CCA) chosen to be analogous yet distinctly vibrant, highlighting key interactive elements and calls to action.
- Primary font: 'Inter' (sans-serif) for all text elements, providing excellent readability, a modern aesthetic, and neutrality suitable for a data-intensive clinical dashboard.
- Use a consistent set of clean, simple, and vector-based icons that align with modern clinical and digital interface standards. Icons should be clear and convey meaning quickly without being distracting.
- Adopt a grid-based, modular layout reminiscent of professional decision-support software. Utilize sidebars for navigation, clear header areas for branding, and well-defined cards or panels for displaying different data points and features, ensuring optimal use of screen space and content hierarchy.
- Implement subtle and purposeful animations for transitions between sections, loading states, and data updates. Avoid flashy or overly dynamic animations to maintain a professional and stable user experience suitable for a healthcare application.
